Zulip server installation

To use Zulip for your team, you will need to install an instance of the Zulip server.

Currently, the automated Zulip server installation process only supports Ubuntu 14.04 Trusty.

You can install a Zulip server on a system with 2G of RAM, but for production use we recommend a system with 4GB of RAM or more.

Install the production server

Download the tarball

Download the binary tarball and extract it to /root/zulip

Install SSL certs

Zulip uses HTTPS so you need to generate SSL keys and install them here:
/etc/ssl/private/zulip-enterprise.key
/etc/ssl/certs/zulip-enterprise.combined-chain.crt

Run the setup script
cd /root/zulip
./scripts/setup/install

This may take a while to run, since it will install a large number of packages via apt.

Configure Zulip

Configure the Zulip server instance by filling in the settings in /etc/zulip/settings.py

Initialize the database

Run the following command:

su zulip -c /home/zulip/deployments/current/scripts/setup/initialize-database
This will report an error if you did not fill in all the mandatory settings from /etc/zulip/settings.py. Once this completes successfully, the main installation process will be complete, and if you are planning on using password authentication, you should be able to visit the URL for your server and register for an account.

Integrations

Once you have the Zulip server running, you can reconfigure various options, auth methods, and integrations in /etc/zulip/settings.py; after making a change you will need to restart the server using:

su zulip -c /home/zulip/deployments/current/scripts/restart-server
