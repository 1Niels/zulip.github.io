About Zulip server

Zulip server is a Linux script that runs the Zulip Python app, Postgres database, and queuing server.

Once up and running, users in your organization can use the web app hosted on your domain, or download a client app from zulip.org

Your best bet right now is running Zulip on Ubuntu 12.04 LTS (Precise Pandolin) on AWS EC2.

A word of warning

Running Zulip in production is a little difficult right now. We're working on cleaning up the production process but please keep this in mind as you work through the following steps!

Install the production server

Download the tarball

Download the binary tarball and extract it to /root/zulip

Install SSL certs

Zulip uses HTTPS so you need to generate SSL keys and install them here:
/etc/ssl/private/zulip-enterprise.key
/etc/ssl/certs/zulip-enterprise.combined-chain.crt

Run the setup script
cd /root/zulip
./scripts/setup/install

This may take a while to run, since it will install a large number of packages via apt.

Configure Zulip

Configure the Zulip server instance by filling in the settings in /etc/zulip/settings.py

Initialize the database

Run the following command:

su zulip -c /home/zulip/deployments/current/scripts/setup/initialize-database
This will report an error if you did not fill in all the mandatory settings from /etc/zulip/settings.py. Once this completes successfully, the main installation process will be complete, and if you are planning on using password authentication, you should be able to visit the URL for your server and register for an account.

Integrations

You can configure additional integrations in /etc/zulip/settings.py

Maintaining your installation

Read through the README in /root/zulip for instructions on upgrading and maintaining your Zulip instance.
